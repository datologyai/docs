A recipe is a named, supported curation template. It defines expected inputs,
output components, budgets, and constraints for a curation. Use the recipe name
when launching a customer curation; the CLI resolves the lower-level execution
target.

List recipes from the CLI runner:

```bash
datology list-curation-recipes
```

## Supported recipes

| Recipe | Intended use |
| --- | --- |
| `web_multilingual_stew` | Web and multilingual curation across quality tiers, with optional per-language translation. |
| `wrapped_stew_001` | Full stew combining web, code, math, and multilingual curation into one deprecated joint ingest-and-curate workflow. |

## `web_multilingual_stew`

Use this recipe for web and multilingual curation without code or math.

Recommended token range:

```text
60B - 10T tokens
```

Expected inputs:

- Web source data
- Web target data for distribution fitting
- Optional multilingual source data with language labels
- Optional multilingual target data

Labeled output components include:

- High-quality score-filtered web data
- Medium-quality web data
- Distribution-fitted web data
- Rephrased synthetic web data
- Per-language translated data

Notes:

- For a target output size, we require at least 2x the input web tokens after
  deduplication. More frequently we suggest a 5-to-1 ratio.
- Multilingual runs require native source tokens and English tokens.
- Rephrasing can use the registered synthetic models staged in the customer artifact bucket.

## `wrapped_stew_001`

Use this recipe when the target output combines web, code, math, and
multilingual sources.

<Note>
`wrapped_stew_001` still uses the deprecated joint ingest and curate flow. It
requires inline ingestion through `ingest_args` instead of separate dataset
registration.
</Note>

Recommended token range:

```text
60B - 10T tokens
```

Expected inputs:

- Web source data
- Web target data for distribution fitting
- Code source data with language labels
- Math source data
- Multilingual source data with language labels

Labeled output components include:

- High-quality, medium-quality, distribution-fitted, and synthetic web data
- BeyondWeb code rephrasing
- Synthetic code rephrasing
- Curated math data
- Per-language translated data

Notes:

- Requires inline ingestion through `ingest_args`.
- Code and multilingual data require provided language labels.
- Math synthesis needs available H100 nodes.

For customer-facing configuration fields such as source documents, target
documents, token budgets, domain dataset names, model overrides, URL filters,
deduplication, and cluster sizing overrides, see
[Extra curation arguments](/reference/extra-curation-args).

## Runtime compatibility

Recipes are versioned. If a recipe is not registered for the runtime version
you selected, register the version with:

```bash
datology upgrade-launchplans --version <runtime-version>
```
