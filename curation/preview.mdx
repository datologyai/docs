title: "Preview a curation"
description: "Validate a curation before launching a workflow run."

---

Preview mode validates inputs and resolves workflow parameters without
launching a curation.

## Human-readable preview

```bash
datology new-curation \
  --run-name preview-web \
  --recipe web_multilingual_stew \
  --output-dataset-path s3://<bucket>/<output-prefix>/ \
  --output-format parquet \
  --extra-curation-args-file curation_args.json \
  --pretty-preview
```

Use `--pretty-preview` when you want an operator-readable summary.

## Full JSON preview

```bash
datology new-curation \
  --run-name preview-web-json \
  --recipe web_multilingual_stew \
  --output-dataset-path s3://<bucket>/<output-prefix>/ \
  --extra-curation-args-file curation_args.json \
  --full-json-preview
```

Use `--full-json-preview` when you want the full resolved workflow parameters.

## Extra workflow arguments

Extra arguments can come from a JSON file:

```bash
datology new-curation \
  --output-dataset-path s3://<bucket>/<output-prefix>/ \
  --recipe web_multilingual_stew \
  --extra-curation-args-file args.json \
  --pretty-preview
```

See [Extra curation arguments](/reference/extra-curation-args) for the
customer-facing JSON fields supported by recipe configuration files.

Or from a JSON string:

```bash
datology new-curation \
  --output-dataset-path s3://<bucket>/<output-prefix>/ \
  --recipe web_multilingual_stew \
  --extra-curation-args-string '{"curation_ratio": 0.8}' \
  --pretty-preview
```

When both are supplied, string values override duplicate keys from the file.

## What preview validates

Preview checks that the CLI can resolve the version, recipe, and workflow
inputs. It does not guarantee that enough cluster capacity exists or that every
downstream data read will succeed.
