title: "Launch a curation"
description: "Start a curation workflow from the CLI."

---

Use `datology new-curation` to launch a curation.

```bash
datology new-curation \
  --run-name web-curation-001 \
  --recipe web_multilingual_stew \
  --output-dataset-path s3://<bucket>/<output-prefix>/ \
  --output-format parquet \
  --extra-curation-args-file curation_args.json
```

The command returns a Curation ID and output folder.

<Note>
`--input-dataset-path` and `--input-schema` are deprecated for curation
launches. Describe input data in `curation_args.json` with the
`IngestionSource` schema instead.
</Note>

## Common options

| Option | Purpose |
| --- | --- |
| `--run-name` | Human-readable run name. |
| `--recipe` | Recipe to execute, such as `web_multilingual_stew` or `wrapped_stew_001`. |
| `--output-dataset-path` | Output path for curated data. |
| `--output-format` | Output format: `parquet`, `mds`, `jsonl`, or `lit`. |
| `--curation-ratio` | Ratio of output size to input size. |
| `--version` | Runtime and recipe version. |
| `--extra-curation-args-file` | JSON file with recipe inputs and `IngestionSource` entries. |
| `--threadpool-max-workers` | Max workers for threadpool execution. |

Use [Extra curation arguments](/reference/extra-curation-args) when building
the JSON passed to `--extra-curation-args-file`.

## Ingestion sources and output formats

Use `IngestionSource` entries to tell the workflow how to read source data.
Each entry names the source, points to its object-store location, and provides
ingestion overrides such as the text column, file type, and metadata mapping.

```json
{
  "source_documents": [
    {
      "name": "fineweb",
      "source_location": "s3://<bucket>/<input-prefix>/",
      "override_args": {
        "text_column": "text",
        "file_type": "parquet"
      }
    }
  ]
}
```

Use `--output-format` to choose the format written to the output path.

- `parquet`: Default columnar output.
- `mds`: MosaicML Streaming format.
- `jsonl`: JSON lines output.
- `lit`: LitData-style output.

Column-level input contracts should be confirmed per source before launching
customer examples that depend on exact fields.

## Version selection

If `--version` is not passed, the CLI uses the configured default version from
`datology_config.json`. If the recipe is not registered at that version, the
CLI can prompt for registration.

```bash
datology new-curation \
  --output-dataset-path s3://<bucket>/<output-prefix>/ \
  --recipe web_multilingual_stew \
  --extra-curation-args-file curation_args.json \
  --version <runtime-version>
```

## Parallel runs

By default, the CLI allows one running curation at a time. To override this for
an approved environment:

```bash
export ALLOW_PARALLEL_CURATION_RUNS=true
```

Use parallel runs only after confirming the cluster has enough capacity.
