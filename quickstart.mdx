This path assumes Datology has provided access to the released deployment
artifacts for your AWS account.

## 1. Prepare AWS access

Set the AWS profile and required deployment inputs.

```bash
export AWS_PROFILE="<customer-aws-profile>"
export DEPLOYMENT_PUBLIC_KEY="<ssh-public-key>"
export CUSTOMER_NAME_PREFIX="<customer-name>"
export VPC_SLASH_16_CIDR_BLOCK="10.50"
```

If you are deploying into an existing VPC or EKS cluster, use the variables in
the [deployment environment reference](/reference/deployment-environment-variables)
instead of `VPC_SLASH_16_CIDR_BLOCK`.

## 2. Deploy the stack

Run the released deployment entry point.

```bash
./deploy_datology.sh
```

The deployment creates the AWS infrastructure, installs the Kubernetes product
stack, creates a CLI runner, and writes the configuration files the CLI uses.

## 3. Connect to the CLI runner

After the stack completes, connect to the CLI runner instance. The runner has
the `datology` wrapper, `kubectl`, `flytectl`, and the generated Datology
configuration.

```bash
datology --help
datology version
datology list-curation-recipes
```

The runner exports the standard CLI environment variables:

```bash
export DATOLOGY_CONFIG_PATH=/home/ubuntu/.config/datology_config.json
export FLYTECTL_CONFIG=/home/ubuntu/.config/flyte_config.yaml
export ASSET_TRACKER_PROD_URI="<asset-tracker-uri>"
export KUBECONFIG=/home/ubuntu/.kube/config
```

## 4. Register a dataset

Register source data from object storage. Dataset registration launches a
workflow run, so wait for it to complete before using the dataset in a
curation.

```bash
datology add-dataset \
  --run-name first-dataset \
  --input-dataset-path s3://<bucket>/<source-prefix>/ \
  --output-dataset-path s3://<bucket>/<registered-prefix>/
```

Inspect registration status:

```bash
datology list-datasets
datology describe-dataset <run_id>
```

## 5. Preview and launch a curation

Preview validates inputs and shows the resolved workflow parameters without
starting a run.

```bash
datology new-curation \
  --run-name first-curation-preview \
  --input-dataset-path s3://<bucket>/<registered-prefix>/ \
  --input-schema dclm_parquet \
  --output-dataset-path s3://<bucket>/<curated-output-prefix>/ \
  --output-format parquet \
  --launch-plan web_multilingual_stew_from_commons_threadpool \
  --pretty-preview
```

Launch the curation after the preview looks correct:

```bash
datology new-curation \
  --run-name first-curation \
  --input-dataset-path s3://<bucket>/<registered-prefix>/ \
  --input-schema dclm_parquet \
  --output-dataset-path s3://<bucket>/<curated-output-prefix>/ \
  --output-format parquet \
  --launch-plan web_multilingual_stew_from_commons_threadpool
```

The command returns a Curation ID and an output folder.

## 6. Monitor and inspect outputs

```bash
datology list-curations
datology show-curation <curation_id>
datology info s3://<bucket>/<curated-output-prefix>/
```

Use `ddb <curation_id>` when `show-curation` is not enough and you need Spark,
Ray, task, or log details.
