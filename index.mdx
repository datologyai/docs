Datology is a platform that turns large volumes of raw data into high-quality datasets
for training AI models. It identifies and shapes the most valuable data so teams can
train better models with less data and compute, reducing the risk of underperforming
high-cost training runs.


These docs how to operate the Datology product, from deploying it to curating datasets.

## Platform overview

Datology runs in customer-controlled infrastructure. Your data stays in your
environment while Datology orchestrates distributed curation workflows against
the object-storage paths and compute resources you configure.

The platform is organized around a few customer-facing primitives:

- **Recipes** define supported curation workflows.
- **Datasets** represent source or produced data in object storage.
- **Curations** run recipes against datasets.
- **Assets** track produced outputs, intermediate data, metadata, and lineage.
- **Versions** bind curations to registered runtime behavior.

The current customer interface is the `datology` CLI on the CLI runner created
by deployment.

## Core functionality

Datology curation combines multiple data-improvement techniques into repeatable
workflows:

- **Quality filtering** separates high-signal examples from noisy or harmful data.
- **Synthetic enhancement** creates useful variations of strong content.
- **Deduplication and redundancy control** remove harmful repetition while preserving useful reinforcement.
- **Task and domain relevance** emphasize data that matches customer training goals.
- **Curriculum and sequencing** organize data so models learn more efficiently.
- **Multilingual curation** extends curation benefits beyond English-only datasets.

The output is a curated dataset plus the metadata needed to understand where it
came from, which run produced it, and which artifacts are associated with it.
