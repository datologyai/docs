Datology has two version concepts and matching upgrade paths. Stack upgrades
change infrastructure. Runtime upgrades change the images and recipe versions
used by curations.

## Stack upgrades

Stack upgrades change infrastructure or Kubernetes services. They may update:

- CloudFormation templates
- EKS resources
- IAM roles or policies
- Product stack Helm templates
- Databases or service configuration
- Installer and CLI runner behavior

Use the released deployment path and Datology-provided version guidance for
stack upgrades.

The stack version identifies the released CloudFormation templates, install
scripts, Kubernetes product stack templates, and CLI artifacts used during
deployment.

Set a specific stack version with:

```bash
export DATOLOGY_STACK_VERSION="<stack-version>"
```

When unset, the deployment resolves the latest released stack version from the
release bucket.

## Runtime upgrades

Runtime upgrades change the images and recipe versions used by curations.
They do not necessarily change the infrastructure stack.

Set a specific runtime image tag with:

```bash
export RUNTIME_CONTAINER_IMAGE_TAG="<runtime-version>"
```

The generated `datology_config.json` stores this as `default_version`. Future
curations use that default unless a command passes `--version`.

Register recipe entry points for a runtime version:

```bash
datology upgrade-launchplans --version <runtime-version>
```

This command:

- Resolves the effective runtime version.
- Registers recipe entry points for that version.
- Updates the configured default version when `--version` is provided.
- Reactivates scheduled recipes for the registered version.

## Required access

Runtime registration requires:

- `DATOLOGY_CONFIG_PATH`
- `FLYTECTL_CONFIG`
- `ASSET_TRACKER_PROD_URI`
- `KUBECONFIG`
- ECR access to the runtime image repository

Treat this as an operator/admin action.

## Identify the version used by a run

Use the output path with `datology info`:

```bash
datology info s3://<bucket>/<asset-path>/
```

The output includes the execution version, execution ID, Run ID, and run name.
