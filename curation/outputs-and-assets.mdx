`new-curation` returns an output folder. The curation writes data there and
records asset metadata in the asset tracker.

## Inspect asset lineage

```bash
datology info s3://<bucket>/<asset-path>/
```

The output includes:

- Execution version
- Execution ID
- Run ID
- Run name

Use `--verbose` to show recorded workflow inputs:

```bash
datology info s3://<bucket>/<asset-path>/ --verbose
```

## Backfill an existing asset

<Note>
Use `backfill` only for importing Datology-supplied artifacts that are already
in known internal formats. Those formats are not fully specified for customer
use. For customer source data, use [dataset registration](/curation/datasets).
</Note>

When Datology support directs you to register a supplied artifact, use
`backfill`:

```bash
datology backfill \
  --input-dataset-path s3://<bucket>/<existing-path>/ \
  --input-dataset-name raw
```

## Keep traceability fields

For important outputs, keep:

- Output S3 path
- Run ID
- Curation ID
- Execution version
- Recipe
- Runtime version

These fields are enough to connect an output path to the workflow and runtime
that produced it.
